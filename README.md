# ⚡️ NUIST 宿舍电量监控组件 (Scriptable)

一个为**南京信息工程大学 (NUIST)** 学生量身定制的 iOS 桌面小组件。基于 [Scriptable](https://scriptable.app/) 开发，能够自动登录统一身份认证系统 (CAS)，穿透一卡通微服务接口，将宿舍实时剩余电量优雅地展示在你的 iPhone 屏幕上。

从此告别打游戏/赶 DDl 时宿舍突然断电的尴尬！🎮 💻

---

## ✨ 核心特性

* 📊 **精美 UI 与电量预警**：采用拟物化的圆角进度条设计。电量状态一目了然：
* 🟢 **健康** (> 50度)：原谅色，安全感满满。
* 🟠 **警告** (10 ~ 50度)：日落橙，提醒你该准备充值了。
* 🔴 **危险** (≤ 10度)：警报红，断电边缘疯狂试探。


* 🤖 **全自动无感登录**：完美适配 NUIST CAS 认证流程，支持通过第三方 OCR 接口自动识别并突破登录验证码。
* 🔐 **本地安全运行**：账号密码仅保存在本地设备的 Scriptable 脚本内，不经过任何中间服务器（除验证码图片交由 OCR 接口识别外）。
* 🔄 **静默刷新**：遵循 iOS 桌面组件的生命周期，默认设置约 30 分钟后台自动刷新一次。

## 🛠️ 安装指南

1. **下载依赖 App**
在 App Store 中搜索并下载 [Scriptable](https://apps.apple.com/cn/app/scriptable/id1405459188)。
2. **导入脚本**
* 打开 Scriptable，点击右上角的 `+` 号新建一个脚本。
* 将本仓库中的 `main.js` 代码全量复制并粘贴进去。
* 点击标题重命名为 `NUIST 电量`，并保存。


3. **添加桌面小组件**
* 回到 iOS 桌面，长按空白处进入编辑模式，点击左上角 `+` 号。
* 找到 Scriptable，选择 **小号 (Small)** 尺寸的组件并添加到桌面。
* 长按该组件选择“编辑小组件”，在 `Script` 选项中选择你刚才保存的 `NUIST 电量`。

## ⚙️ 配置说明

在代码的最顶端（第 5 行开始），有一个 `CONFIG` 配置区，你必须将其修改为你自己的信息才能正常运行：

```javascript
const CONFIG = {
    // 1. 账号密码 (学校统一身份认证门户的账号密码)
    username: "你的学号",
    password: "你的密码",

    // 2. 宿舍具体参数 (需要抓包获取)
    type: "IEC",
    level: "3",
    feeitemid: "448",              // 业务线：本部填448，天长校区填429
    xiaoyu_id: "3&沁园",           // 你的校区ID及名称
    loudong_id: "23&沁园30号栋",   // 你的楼栋ID及名称
    room_id: "4186&135",         // 你的房间ID及名称

    // 3. OCR 验证码识别接口
    ocrApiUrl: "http://你的服务器IP:端口/classification", 
    
    // 4. UI 视觉阈值 (默认即可，也可根据寝室耗电情况微调)
    ROOM_CAPACITY: 100,     // 进度条满量程
    WARNING_THRESHOLD: 50,  // 变橙色阈值
    DANGER_THRESHOLD: 10,   // 变红色阈值
};

```

### ❓ 如何获取宿舍的具体参数？

由于学校接口采用 `ID&名称` 的级联查询逻辑，你需要通过浏览器控制台（F12）或抓包工具（如 Stream / Reqable）在充值电费页面抓包获取。

1. 登录学校电脑端一卡通充值大厅。
2. 按 `F12` 打开开发者工具，切换到 `Network` (网络) 面板。
3. 在页面上依次选择你的 **校区 -> 楼栋 -> 房间**。
4. 观察名为 `getThirdData` 的网络请求，查看其 `Payload` (请求负载) 中的表单数据，将对应的值填入 `CONFIG` 中即可。


## 🤖 验证码识别 (OCR) 部署指南

由于学校的统一身份认证 (CAS) 强制要求输入图形验证码，本脚本依赖第三方 OCR 接口进行自动识别。我们强烈推荐使用开源的 [sml2h3/ddddocr](https://github.com/sml2h3/ddddocr) 方案。

为了方便大家，这里提供最快速的 Docker 部署方式（你需要有一台云服务器或软路由/NAS）：

### 一键部署 OCR API
你可以使用社区封装好的 ddddocr API 镜像，只需在你的服务器终端运行以下命令：

```bash
# 拉取并运行 ddddocr 的 Web API 容器，映射到宿主机的 7777 端口
docker run -d -p 7777:8000 --name ddddocr-api mzzsfy/ddddocr:latest
```
